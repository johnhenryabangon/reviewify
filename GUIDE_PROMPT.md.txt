# Claude Development Guide for Reviewify

You are helping develop a modern full-stack Python web application called:

# Reviewify
An AI-Powered Automated Reviewer Generation System

---

# Development Goal

Build a clean, scalable, and beginner-friendly automation system that:
1. accepts PPT/PPTX/PDF lecture uploads,
2. extracts lecture content,
3. summarizes and organizes topics,
4. automatically generates reviewer documents,
5. exports reviewers into PDF and DOCX formats.

The project should feel like a real-world educational productivity SaaS platform.

---

# Tech Stack

## Backend
- Python
- Flask
- SQLite

## Frontend
- HTML
- TailwindCSS
- Vanilla JavaScript

## File Processing
- python-pptx
- PyMuPDF or pdfplumber

## AI/NLP
- transformers
- summarization models

## Document Generation
- python-docx
- reportlab

# Development Instructions

## Code Quality
- Write clean and maintainable code
- Use modular architecture
- Separate logic into utility files
- Add comments for important functions
- Follow beginner-friendly structure

## UI/UX Requirements
The interface should look modern and polished.

Design inspiration:
- Notion
- Canva Docs
- modern AI SaaS dashboards

UI requirements:
- responsive layout
- dark/light mode
- minimalist dashboard
- glassmorphism cards
- smooth animations
- clean academic appearance

# Core Modules

## Upload Module
- multi-file upload
- drag-and-drop support
- file validation

## Extraction Module
- extract slide text
- detect headings/topics
- preserve formatting when possible

## Summarization Module
- summarize lengthy content
- generate concise reviewer notes

## Reviewer Formatting Module
- generate two-column reviewer layout
- optimize spacing for printing

## Export Module
- generate PDF
- generate DOCX

---

# Important Requirements

- Ensure generated reviewer is printable
- Prioritize readability
- Keep processing efficient
- Add proper error handling
- Add loading states while processing
- Ensure maintainable architecture

---

# Preferred Workflow

When generating code:
1. start with backend structure,
2. then upload functionality,
3. then extraction logic,
4. then summarization,
5. then reviewer generation,
6. finally UI polishing.

---

# Expected Output Style

When generating code:
- provide complete files
- specify filenames clearly
- explain where files should be placed
- include setup instructions
- avoid overly complex architecture
- prioritize working implementation first

---

# Main Objective

The final project should automate the tedious process of manually creating reviewer documents from lecture slides and significantly improve students' study workflow.
